Component Library

Every reusable block for dwellcc.org, pulled from the live pages — real markup, real CSS, real responsive breakpoints. Preview a component, flip to Code, copy, paste. This library lives on the Dwell site, so components inherit the theme’s Circe/Lato fonts, jQuery, and GSAP.

Colors #

The real palette used across the site: dark teal surfaces, leaf & lime greens for action, plus the badge tints from the resources page.

Typography #

Circe is the primary font across every component; weight 900 for headings, 400/700 for body.

Buttons #

The real .btn-dwell-primary (used for “Add Resource”) and the dark resource CTA pill. Both from the leaders page.

Section eyebrow #

The small uppercase green label that opens most content sections (Circe 900, 2px tracking, #07841B).

Hero band #

Dark teal page header with faint decorative circles. Responsive: padding & type shrink under 768px.

Member notice #

Thin deep-green strip below the hero prompting members to log in.

Page title bar #

The on-page title/subnav bar. Real responsive behaviour: a gray bar with right-aligned links at ≥1200px; below that it collapses to a tappable green bar that expands the nav (GSAP). Needs jQuery + GSAP (loaded site-wide).

Feature cards #

Two-up promo cards with a tag, heading, description, and arrow CTA. Collapses to one column under 768px.

Featured events #

Gradient banner with a “Register now” tag and a 3-up row of event tiles. Grid goes 3→2→1. Note: on the live site this uses the dwl-feat- prefix; renamed here to dwl-events- so it can coexist with the content-list rows, which also use dwl-feat-.

Image callout cards #

Photo card with a badge overlay, title, description, and CTA. Two-up, collapsing to one column under 768px. (Real Cloudinary image URLs kept in.)

Link cards #

Compact cards with a heading, description, and a dot-separated group of links. Two-up → one column under 768px.

Icon cards #

Icon-plus-text cards in a 3-up grid (3→2→1). Uses Font Awesome Light icons (loaded site-wide).

Explore-interest cards #

Green left-border cards for browsing by interest. First card can span two columns (--wide). Grid steps 5→3→2→1 across breakpoints.

Resource-type cards #

Lime left-border cards for content types. 4-up grid collapsing to 2-up under 768px.

Content list rows #

Divider-separated rows with a colored type badge, title, optional meta, and optional date. Badge tints: essay/teaching/podcast green, tool/class tan, story teal. Rows reflow on mobile.

Topic tiles #

Section header (title + count + rule) over an auto-fill grid of topic cards with a hover-reveal left accent and resource count. Tightens under 640px.

Giving info cards #

Two-up cards with a heading, description, and arrow CTA; lime hover border. Collapses to one column under 768px.

Split link rows #

Divider-separated rows with info on the left and an action link on the right. Used for financial transparency.

Scripture quote #

Full-bleed dark band with an oversized quote mark, italic verse, and attribution. Uses width:100vw + negative margins to break out of the content column full-width.

Filter panel #

Dark browse sidebar: search + category/type/sort selects with active state & clear-X, plus clear-all. Real responsive behaviour: below 900px it becomes a fixed bottom sheet (see the Resources app on the live leaders page). Shown here in its desktop shell; wired with htmx + GSAP on-site.

Grouped resource list #

Category-grouped results: uppercase section label with trailing rule, a bordered white card of rows (green icon tile + title + meta + summary), and a Show More control (htmx-driven on-site). Rows tighten under 560px.

Resource CTA button #

Centered dark pill that lifts and turns lime on hover.

Site header #

The authentic global top bar — dark teal with a lime underline, logo, nav links, and utility icons (translate, search, account dropdown). Below 768px the inline nav swaps to a full-width row. Ships with its stylesheet and the dropdown/keyboard JS (needs jQuery; the Google-CSE search overlay lives in a separate block).

Footer #

The real global footer markup (.site-footer). It relies on the theme’s Bootstrap grid + footer styles, which are loaded everywhere on dwellcc.org — so it renders correctly on-site (this preview lacks theme.css, so it appears unstyled here).
